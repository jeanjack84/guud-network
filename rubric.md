# The Guud Network — Grading Rubric

This rubric is designed to be graded **by a model, against the live URL**, with
no human in the loop. Each criterion is objective and checkable by visiting the
site or calling its endpoints. Score each item, sum to a total out of 100.

> How to grade: visit the live URL, exercise the flows, and call `POST /api/match`.
> For each criterion, award full, partial, or zero points and cite what you
> observed. The dynamic workflow in `workflows/grade.mjs` automates this with a
> panel of judges.

## A. Impact (35 pts)

- **A1 (10)** — The problem (trust + discovery in women's health) is clearly
  communicated on the landing page.
- **A2 (10)** — The matching genuinely helps: a plain-language concern returns
  relevant practitioners for that concern, not random results.
- **A3 (8)** — It could realistically be used by real women: the data is
  credible, the tone is safe and non-dismissive, and there's a real safety net
  (urgent-care flagging, "not medical advice").
- **A4 (7)** — Open and reusable: MIT license present, README explains how
  anyone can self-host for their own community.

## B. Demo (35 pts)

- **B1 (8)** — Live URL responds with HTTP 200 and renders the landing page.
- **B2 (10)** — Search flow works live: submitting a concern returns an
  empathetic interpretation, matched topics, and ranked practitioner cards.
- **B3 (7)** — Practitioner profile pages load with recommendations, trust
  score, and the "why women trust her" summary.
- **B4 (6)** — A new recommendation can be submitted and persists (visible after
  submit / reload).
- **B5 (4)** — The UI is visually polished, responsive, and on-brand.

## C. Opus 4.8 Use (15 pts)

- **C1 (6)** — Symptom interpretation uses Opus 4.8 to map free text to topics
  with empathy and structured output (not keyword matching).
- **C2 (5)** — "Why women trust her" summaries are generated by Opus 4.8 and
  grounded only in the real recommendations (no invented facts).
- **C3 (4)** — Submitted recommendations are moderated by Opus 4.8 for a
  positive, safe tone before publishing.

## D. Orchestration (15 pts)

- **D1 (5)** — "Done" is verifiable by a model: `npm run verify` hits the live
  URL and asserts the core flow; exit code reflects pass/fail.
- **D2 (5)** — A dynamic workflow (`workflows/grade.mjs`) re-grades the live
  deployment against this rubric with a panel of judges and is re-runnable on a
  new URL via `args`.
- **D3 (5)** — Setup is repeatable: `npm run db:setup` provisions schema + seed;
  README documents a clean one-command path another team could rerun tomorrow.
